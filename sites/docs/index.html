Overview

Death2Data is a privacy-first metasearch engine that aggregates results from 70+ search engines without tracking you. No IP logging. No query storage. No profiling. Ever.

Currently in development. Join the waitlist to get notified when we launch.

Quick Start

The fastest way to get Death2Data running locally:

docker run -d -p 8080:8080 death2data/search:latest

Then open http://localhost:8080 in your browser.

Installation

Death2Data can be installed in several ways:

  • Docker — Recommended for most users
  • Docker Compose — For production deployments
  • Manual — For advanced customization

Requirements

  • Docker 20.10+ (for containerized deployment)
  • Python 3.9+ (for manual installation)
  • 512MB RAM minimum (1GB+ recommended)

Settings

Death2Data is configured via environment variables or a settings.yml file.

# Basic settings.yml example
general:
  debug: false
  instance_name: "My Death2Data"

search:
  safe_search: 0
  default_lang: "en"
  
server:
  port: 8080
  bind_address: "0.0.0.0"

Search Engines

Death2Data aggregates results from multiple sources. By default, the following engines are enabled:

  • DuckDuckGo
  • Brave Search
  • Qwant
  • Wikipedia
  • And 70+ more...

You can enable/disable individual engines in the settings or through the web UI.

Privacy Options

Death2Data is built with privacy as the default:

  • No logging — Search queries are never stored
  • No tracking — No cookies, fingerprinting, or analytics
  • Proxy images — Images are proxied to prevent tracking pixels
  • Remove trackers — URL tracking parameters are stripped

Note: While Death2Data doesn't track you, the upstream search engines may. For maximum privacy, use Death2Data over Tor.

Docker Deployment

Using Docker Compose for a production-ready setup:

# docker-compose.yml
version: '3'
services:
  death2data:
    image: death2data/search:latest
    container_name: death2data
    restart: unless-stopped
    ports:
      - "8080:8080"
    volumes:
      - ./settings.yml:/etc/death2data/settings.yml:ro
    environment:
      - BASE_URL=https://search.yourdomain.com

Run with: docker-compose up -d

Manual Setup

For manual installation:

# Clone the repository
git clone https://github.com/deathtodata/search.git
cd search

# Install dependencies
pip install -r requirements.txt

# Run the server
python death2data.py

Reverse Proxy

Example Nginx configuration for running behind a reverse proxy:

server {
    listen 443 ssl http2;
    server_name search.yourdomain.com;

    location / {
        proxy_pass http://localhost:8080;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
